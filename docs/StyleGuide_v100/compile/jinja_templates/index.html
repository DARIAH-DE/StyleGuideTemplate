Purpose

The purpose of the DARIAH-DE StyleGuideTemplate (SGT from now on) is to provide a basic template for all web applications that are part of the DARIAH-DE ecosystem.

Tip: If you are about to start developing a new DARIAH-DE application, by following the SGT the look & feel of your app is going to be consistent with the rest of the applications.
Tip: If you have already developed a DARIAH-DE application, you may want to adjust its layout so as to match the one described by the SGT.

Dependencies

This template relies on the following external resources:

  • Boostrap Framework (v. 2.2)
  • jQuery Library (v. 1.8.2)
  • Font Awesome icons (v. 4.3.0), e.g. etc.
  • FamFamFam flag icons, e.g. etc.

How to use this template?

Clone the github repository and start adapting the template to your own application (e.g. integration with the template engine of your choice).

What to change

  • the tool name
  • the "breadcrumbs" section
  • the list of supported languages
    Tip: To find the language list look for this element in the HTML source code: 
    <ul id="language_selection">...</ul>
  • the licence information
    Tip: Look for this element in the HTML source code: 
    <div id="footer">...</div>

What not to change

The DARIAH-DE Global Menu must remain unchanged (i.e. the dropdown menu in the top-left corner which is re-populated from a global JSON definition).

Content

Templates

The /templates/ directory contains the following template variants:

  • /templates/notloggedin.html : this template assumes that the user is not logged in as indicated by the icon displayed in the top-right corner
  • /templates/loggedin.html : this template assumes that the user is logged in as indicated by the icon displayed in the top-right corner
  • /templates/withlogo.html : this is an alternate template that displays the DARIAH-DE logo within the dropdown menu in the top-left corner

Resources

The /resources/ directory contains all the files required by the SGT in order to display correctly.

This directory has the following straight-forward structure:

  • /resources/css/
  • /resources/fonts/
  • /resources/img/
  • /resources/js/

If you move the resources directory make sure you change the path of links pointing to its content (e.g. js, css, icons, etc.).
